## Setup Instructions

### Step 1: Create a Notion Integration

1.  Go to the [Notion My Integrations page](https://www.notion.so/my-integrations) and click **"New integration"**.
2.  Give your integration a name (e.g., "Teta AI").
3.  Select the workspace where you want to install the integration.
4.  For capabilities, select **"Read content"**. Write capabilities are not yet supported.
5.  Click **"Submit"**.

### Step 2: Get the Integration Token

1.  On the next screen, you'll see your **"Internal Integration Token"**. Copy this token.

### Step 3: Share Your Pages/Databases with the Integration

1.  Go to the Notion page or database you want to share with Teta.
2.  Click the **"â€¢â€¢â€¢"** menu in the top-right corner.
3.  Click **"Add connections"** (or **"Connections"** > **"Connect to"**).
4.  Find and select your new integration (e.g., "Teta AI").
5.  Confirm that you want to grant access.

### Step 4: Add Token to Teta

1.  In your Teta project, go to **Settings** > **MCPs**.
2.  Click **"Add MCP"**.
3.  Add a name for the secret, for example `server-notion`.
4.  Add the following JSON configuration, replacing `<YOUR_NOTION_INTEGRATION_TOKEN>` with the token you copied.

```json
{
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-notion"],
  "env": {
    "NOTION_API_KEY": "$NOTION_TOKEN"
  }
}
```
5. In the Secrets section, add a new secret named `NOTION_TOKEN` and paste your Internal Integration Token.

## Security Best Practices

### 1. Limit Access

- Only share the specific Notion pages and databases that Teta needs to access.
- Avoid sharing your entire workspace if possible.

### 2. Use Read-Only Tokens

- The current Notion MCP server only supports read access, which is a good security practice.
- When write support is added, use separate tokens for read-only and write operations if possible.

### 3. Protect Your Token

- Treat your Notion Integration Token as a secret.
- Never commit it directly to your repository.
- Use Teta's secret management to store it securely.
