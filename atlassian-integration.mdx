## Setup Instructions

### Step 1: Create an Atlassian API Token

1.  Log in to your Atlassian account.
2.  Go to [Atlassian API Tokens](https://id.atlassian.com/manage-profile/security/api-tokens).
3.  Click **"Create API token"**.
4.  Give the token a descriptive **Label** (e.g., "Teta AI").
5.  Copy the generated token. **You will not be able to see it again.**

### Step 2: Add Token to Teta

1.  In your Teta project, go to **Settings** > **MCPs**.
2.  Click **"Add MCP"**.
3.  Add a name for the secret, for example `server-atlassian`.
4.  Add the following JSON configuration. You will need to provide your Atlassian email and the API token you just created.

```json
{
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-atlassian"],
  "env": {
    "ATLASSIAN_USERNAME": "$ATLASSIAN_EMAIL",
    "ATLASSIAN_API_TOKEN": "$ATLASSIAN_TOKEN",
    "ATLASSIAN_BASE_URL": "https://<your-domain>.atlassian.net"
  }
}
```
5. In the Secrets section, add the following secrets:
    - `ATLASSIAN_EMAIL`: Your Atlassian account email.
    - `ATLASSIAN_TOKEN`: The API token you generated.
    - Replace `<your-domain>` with your actual Atlassian domain.

## Security Best Practices

### 1. Use a Service Account

- If possible, create a dedicated service account in Atlassian for Teta.
- This avoids using a personal account and makes it easier to manage permissions.

### 2. Limit Permissions

- When creating the API token, ensure the associated account has the minimum necessary permissions.
- For example, if you only need to read from Confluence, use an account that doesn't have write access.

### 3. Protect Your Token

- Treat your Atlassian API Token as a secret.
- Never commit it directly to your repository.
- Use Teta's secret management to store it securely.
